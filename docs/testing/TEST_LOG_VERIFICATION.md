# 测试日志验证指南

## 📋 测试前准备

### 1. 确保日志输出可见

在测试时，确保你能看到控制台日志输出：

```bash
# Flutter 运行命令
flutter run

# 或使用 verbose 模式查看更多日志
flutter run -v
```

### 2. 日志级别说明

| 级别 | 图标 | 用途 | 示例 |
|------|------|------|------|
| **DEBUG** | 🐛 | 开发调试信息 | 变量值、流程分支 |
| **INFO** | ℹ️ | 正常业务流程 | 操作开始/成功 |
| **WARNING** | ⚠️ | 警告信息 | 非致命错误、边界条件 |
| **ERROR** | ❌ | 错误信息 | 操作失败、异常 |
| **CRITICAL** | 🔥 | 严重错误 | 应用崩溃、数据丢失 |

## 🧪 绑定功能日志测试

### 测试 1：初始化 BindingProvider

**操作步骤**：
1. 启动应用
2. 进入"绑定"Tab

**预期日志**：
```
ℹ️ INFO   [BINDING] BindingProvider 初始化
ℹ️ INFO   [BINDING] 加载绑定列表
ℹ️ INFO   [BINDING] 绑定列表加载成功
  数据: {total: 0, as_sender: 0, as_receiver: 0}
```

**验证点**：
- ✅ 看到 "BindingProvider 初始化" 日志
- ✅ 看到 "加载绑定列表" 日志
- ✅ 看到 "绑定列表加载成功" 日志，包含数量信息

---

### 测试 2：生成绑定码

**操作步骤**：
1. 点击"绑定新用户"按钮
2. 选择"生成绑定码"
3. 可选：输入备注
4. 点击"生成"按钮

**预期日志（成功）**：
```
ℹ️ INFO   [BINDING] 生成绑定码
  数据: {receiver_phone: null, remark: 给父亲的绑定}
ℹ️ INFO   [BINDING] 绑定码生成成功
  数据: {bind_code: A1B2C3D4}
ℹ️ INFO   [BINDING] 加载绑定列表
ℹ️ INFO   [BINDING] 绑定列表加载成功
  数据: {total: 1, as_sender: 1, as_receiver: 0}
```

**预期日志（失败 - 达到上限）**：
```
⚠️ WARNING [BINDING] 生成绑定码失败: 绑定人数已达上限（最多5个接收者）
```

**预期日志（失败 - 未登录）**：
```
⚠️ WARNING [BINDING] 生成绑定码失败: 未初始化 API 服务，请先调用 init() 方法
```

**验证点**：
- ✅ 看到 "生成绑定码" 日志，包含参数信息
- ✅ 成功时看到 "绑定码生成成功" 和绑定码
- ✅ 失败时看到 "生成绑定码失败" 和错误原因

---

### 测试 3：输入绑定码绑定

**操作步骤**：
1. 点击"绑定新用户"按钮
2. 选择"输入绑定码"
3. 输入绑定码（如：A1B2C3D4）
4. 点击"确认绑定"

**预期日志（成功）**：
```
ℹ️ INFO   [BINDING] 确认绑定
  数据: {bind_code: A1B2C3D4}
ℹ️ INFO   [BINDING] 绑定成功
ℹ️ INFO   [BINDING] 加载绑定列表
ℹ️ INFO   [BINDING] 绑定列表加载成功
  数据: {total: 1, as_sender: 0, as_receiver: 1}
```

**预期日志（失败 - 绑定码无效）**：
```
ℹ️ INFO   [BINDING] 确认绑定
  数据: {bind_code: INVALID}
⚠️ WARNING [BINDING] 绑定失败
  数据: {message: 绑定码无效或已过期}
```

**预期日志（失败 - 异常）**：
```
ℹ️ INFO   [BINDING] 确认绑定
  数据: {bind_code: A1B2C3D4}
❌ ERROR  [BINDING] 确认绑定异常
  数据: {error: NetworkException: 网络连接失败}
  堆栈跟踪: ...
```

**验证点**：
- ✅ 看到 "确认绑定" 日志，包含绑定码
- ✅ 成功时看到 "绑定成功"
- ✅ 失败时看到 "绑定失败" 和错误信息
- ✅ 异常时看到 "确认绑定异常" 和堆栈跟踪

## 📊 日志验证检查清单

### BindingProvider 日志覆盖

- [ ] `init()` - 初始化日志
- [ ] `loadBindings()` - 加载开始/成功/失败日志
- [ ] `generateBindCode()` - 生成开始/成功/失败/异常日志
- [ ] `confirmBinding()` - 确认开始/成功/失败/异常日志
- [ ] `revokeBinding()` - 解除开始/成功/失败/异常日志
- [ ] `checkBindCode()` - 检查开始/成功/失败/异常日志
- [ ] 早期返回分支（未登录、达到上限）有 warning 日志
- [ ] 所有异常都有 error 级别日志和堆栈跟踪

### 日志格式验证

- [ ] 日志标签正确显示 `[BINDING]`
- [ ] 日志级别正确（INFO/WARNING/ERROR）
- [ ] 附加数据正确显示
- [ ] 堆栈跟踪在 error 时显示
