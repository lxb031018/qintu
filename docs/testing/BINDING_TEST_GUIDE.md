### 1. 确保后端已部署
```bash
# 检查云函数是否可访问
curl https://qintu-cloudebase-5f5bpuj13bc6467.service.tcloudbase.com/qintu-api/api/users/me
```

### 2. 确保数据库已初始化
- users 表已创建
- user_bindings 表已创建

### 3. 运行 Flutter 应用
```bash
flutter run
```

## 🧪 测试步骤

### 测试 1：用户登录

1. 打开应用，进入登录页面
2. 输入手机号和验证码
3. 点击登录
4. **预期结果**：登录成功，进入主页

> ⚠️ 注意：当前登录页面还是 TODO 状态，可以使用模拟数据或临时 openid 登录

### 测试 2：查看绑定列表（空状态）

1. 登录成功后，点击底部"绑定"Tab
2. 查看页面显示
3. **预期结果**：
   - 显示统计卡片（0/5 发送者，0/3 接收者）
   - 显示"暂无绑定关系"空状态提示
   - 显示"绑定新用户"按钮

### 测试 3：生成绑定码

1. 点击"绑定新用户"按钮
2. 在弹出的对话框中选择"生成绑定码"
3. 可选：输入备注（如"给父亲的绑定"）
4. 点击"生成"按钮
5. **预期结果**：
   - 显示 8 位绑定码（如：A1B2C3D4）
   - 大字体、蓝色、加粗显示
   - 提示"对方输入此绑定码后即可建立绑定关系"

### 测试 4：输入绑定码绑定

**需要两个设备或账号测试**

#### 账号 A（发送者）
1. 按照"测试 3"生成绑定码
2. 记录绑定码：`XXXX1234`

#### 账号 B（接收者）
1. 登录后进入"绑定"Tab
2. 点击"绑定新用户"
3. 选择"输入绑定码"
4. 输入账号 A 生成的绑定码：`XXXX1234`
5. 点击"确认绑定"
6. **预期结果**：
   - 显示"绑定成功"提示
   - 绑定列表显示新的绑定关系
   - 统计卡片更新（发送者 1/5，接收者 1/3）

#### 账号 A（发送者）验证
1. 刷新绑定列表
2. **预期结果**：
   - 显示与账号 B 的绑定关系
   - 状态为"生效中"（绿色）

### 测试 5：解除绑定

1. 在绑定列表中，找到要解除的绑定
2. 点击右侧的删除图标（红色）
3. 在确认对话框中点击"解除绑定"
4. **预期结果**：
   - 显示"解除绑定成功"提示
   - 绑定从列表中移除
   - 统计卡片数量减少

### 测试 6：绑定人数限制

#### 发送者限制（5 个）
1. 使用账号 A 连续生成 5 个绑定码
2. 使用 5 个不同的接收者账号分别确认绑定
3. 尝试生成第 6 个绑定码
4. **预期结果**：
   - 统计卡片显示"作为发送者：5/5"
   - 统计卡片显示"已达上限"警告
   - "绑定新用户"按钮禁用或点击后提示"绑定人数已达上限"

#### 接收者限制（3 个）
1. 使用账号 B 连续被 3 个发送者绑定
2. 第 4 个发送者尝试绑定
3. **预期结果**：
   - 绑定失败
   - 提示"该用户已被 3 个发送者绑定，无法继续绑定"

### 测试 7：刷新功能

1. 在绑定列表页面
2. 点击 AppBar 的刷新图标
3. **预期结果**：
   - 显示加载指示器
   - 重新加载绑定列表
   - 显示"刷新成功"提示

### 测试 8：错误处理

#### 网络错误
1. 关闭网络连接
2. 尝试加载绑定列表
3. **预期结果**：
   - 显示错误视图（红色错误图标）
   - 显示错误信息
   - 显示"重试"按钮

#### 无效绑定码
1. 输入无效的绑定码（如：不存在的码、已过期的码）
2. 点击"确认绑定"
3. **预期结果**：
   - 绑定失败
   - 显示错误提示（如"绑定码无效"或"绑定码已过期"）

### 测试 9：状态显示

创建不同状态的绑定关系，验证状态显示：

| 状态 | 颜色 | 说明 | 如何触发 |
|------|------|------|----------|
| active | 绿色 | 生效中 | 正常绑定 |
| pending | 橙色 | 待确认 | 生成绑定码后，接收者未确认 |
| expired | 灰色 | 已过期 | 绑定码超过 24 小时 |
| revoked | 红色 | 已解除 | 解除绑定后 |

## 🐛 常见问题排查

### 问题 1：绑定列表加载失败

**可能原因**：
- 后端未部署或无法访问
- 用户未登录
- API 路径错误

**排查步骤**：
1. 检查网络连接
2. 检查 `Constants.baseUrl` 是否正确
3. 检查 `UserProvider.apiService` 是否已初始化
4. 查看控制台日志

### 问题 2：生成绑定码失败

**可能原因**：
- 发送者绑定人数已达上限
- 后端接口错误

**排查步骤**：
1. 检查统计卡片的发送者数量
2. 查看错误提示信息
3. 检查后端日志

### 问题 3：确认绑定失败

**可能原因**：
- 绑定码无效或已过期
- 接收者被绑定人数已达上限
- 后端接口错误

**排查步骤**：
1. 确认绑定码正确且未过期
2. 检查接收者的绑定数量
3. 查看错误提示信息
