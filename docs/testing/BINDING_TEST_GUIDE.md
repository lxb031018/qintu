# 关系绑定系统 - 自动化测试指南

## 📋 测试内容

自动化测试脚本会验证以下流程：

1. ✅ 发送绑定请求
2. ✅ 查看待确认请求
3. ✅ 确认绑定
4. ✅ 查看绑定列表（发送者/接收者）
5. ✅ 重复绑定被拒绝
6. ✅ 绑定自己被拒绝
7. ✅ 未注册手机号返回 404
8. ✅ 解除绑定
9. ✅ 解除后列表为空
10. ✅ 无权解绑他人绑定

## 🚀 快速开始

### 步骤 1: 执行数据库迁移（如果需要）

```bash
# 登录 MySQL
mysql -u your_user -p your_database

# 执行迁移
source D:\AllCodes\qintu\database\migrate_2026_04_09.sql;
```

### 步骤 2: 启动后端服务

```bash
cd D:\AllCodes\qintu\functions\qintu-api

# 安装依赖（如果未安装）
npm install

# 启动开发服务器
npm run dev
```

服务将在 `http://localhost:3000` 启动。

### 步骤 3: 运行自动化测试

打开**新终端**，运行：

```bash
cd D:\AllCodes\qintu\functions\qintu-api
node test-binding-flow.js
```

## ⚠️ 注意事项

1. **数据库必须运行**：测试会读写数据库
2. **Mock 模式**：当前使用 mock token，不需要真实认证
3. **清理数据**：测试会创建测试数据，运行后可手动清理

## 🐛 故障排查

### 服务启动失败

```bash
# 检查端口是否被占用
netstat -ano | findstr :3000

# 检查数据库连接
node -e "require('./lib/database').query('SELECT 1').then(() => console.log('OK'))"
```

### 测试失败

1. 确保服务在 `localhost:3000` 运行
2. 检查数据库连接
3. 查看测试脚本输出的具体错误信息

## 📝 测试脚本说明

- **文件位置**：`functions/qintu-api/test-binding-flow.js`
- **技术栈**：Node.js 原生 HTTP 模块（无需额外依赖）
- **测试数据**：使用 mock openid，不需要真实用户
- **运行时间**：约 2-5 秒
