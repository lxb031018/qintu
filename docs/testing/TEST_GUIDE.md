# 测试指南

## 🧪 绑定功能测试

### 测试前准备

1. 确保后端已部署并可访问
2. 确保数据库已初始化（users 表、user_bindings 表）
3. 运行 Flutter 应用：`flutter run`

### 测试步骤

#### 1. 用户登录
- 输入手机号和验证码，点击登录
- 预期：登录成功，进入主页

#### 2. 绑定列表（空状态）
- 点击底部"绑定"Tab
- 预期：显示统计卡片（0/5 发送者，0/3 接收者）、空状态提示、"绑定新用户"按钮

#### 3. 输入手机号绑定
- 点击"绑定新用户" → 选择"输入手机号绑定"
- 输入对方 11 位手机号
- 预期：绑定成功，列表显示新绑定关系

#### 4. 解除绑定
- 在绑定列表中点击删除图标 → 确认解除
- 预期：绑定从列表移除

#### 5. 绑定人数限制
- 发送者上限：5 个接收者
- 接收者上限：3 个发送者
- 达到上限时，操作应被拒绝并提示

#### 6. 错误处理
- 断网后尝试操作：应显示错误视图和重试按钮
- 输入无效手机号：应提示"手机号无效或用户不存在"

## 🔍 日志验证

### 查看日志

```bash
flutter run
# 或详细模式
flutter run -v
```

### 日志级别

| 级别 | 用途 | 示例 |
|------|------|------|
| DEBUG 🐛 | 开发调试 | 变量值、流程分支 |
| INFO ℹ️ | 正常流程 | 操作开始/成功 |
| WARNING ⚠️ | 警告 | 非致命错误、边界条件 |
| ERROR ❌ | 错误 | 操作失败、异常 |
| CRITICAL 🔥 | 严重错误 | 应用崩溃 |

### 绑定功能关键日志

| 操作 | 预期日志 |
|------|---------|
| 初始化 | `ℹ️ [BINDING] BindingProvider 初始化` |
| 加载列表 | `ℹ️ [BINDING] 绑定列表加载成功` |
| 确认绑定 | `ℹ️ [BINDING] 绑定成功` |
| 解除绑定 | `ℹ️ [BINDING] 解除绑定成功` |
| 失败场景 | `⚠️ [BINDING] xxx失败: 原因` 或 `❌ [BINDING] xxx异常` |

## 🐛 常见问题

### 绑定列表加载失败
1. 检查网络连接
2. 检查 `Constants.baseUrl` 是否正确
3. 检查 `AuthStateManager.state.userId` 是否已设置
4. 查看控制台日志

### 绑定失败
1. 确认手机号正确且对方已注册
2. 检查接收者的绑定数量是否已达上限（3 个）
3. 查看错误提示
