# BindingTab 功能说明

> 本文档描述 BindingTab 页面的 UI 功能和交互逻辑。绑定人数限制规则请参考 [binding_limits.md](./binding_limits.md)。

---

## 📱 页面结构

```
┌─────────────────────────────┐
│  绑定管理           [刷新]  │  ← AppBar
├─────────────────────────────┤
│  当前绑定                    │
│  ┌─────────┬─────────┐     │
│  │ 发送者  │ 接收者  │     │  ← 统计卡片
│  │  2/5    │  1/3    │     │
│  └─────────┴─────────┘     │
├─────────────────────────────┤
│  ┌─────────────────────┐   │
│  │ 👤 父亲      ✓ 生效 │   │
│  │    接收者  [备注]   │   │  ← 绑定列表
│  └─────────────────────┘   │
├─────────────────────────────┤
│  [+ 绑定新用户]             │  ← 添加按钮
└─────────────────────────────┘
```

### 顶部：绑定统计卡片
- 显示当前绑定关系数量（X/5 发送者，X/3 接收者）
- 达到上限时显示红色警告提示
- 实时刷新显示最新数据

### 中部：绑定关系列表
- 显示所有绑定关系
- 每条记录：对方昵称/备注、角色标签、状态标签
- 支持滑动删除（解除绑定）

### 底部：操作按钮
- **绑定新用户**：发送绑定请求
- 达到上限时自动禁用

---

## 🔧 手机号绑定流程

### UI 交互
1. 点击"绑定新用户"按钮
2. 弹出底部表单
3. 输入 11 位手机号（带格式验证）
4. 可选输入备注
5. 提交后显示加载状态
6. 成功后自动刷新列表

### 表单验证规则
- 手机号必须为 11 位数字
- 备注名最多 20 个字符

---

## 🔄 使用流程

### 发送绑定请求
1. 输入对方的 11 位手机号
2. 可选：输入备注
3. 点击"发送绑定请求"
4. 等待接收者确认

### 解除绑定
1. 在绑定列表中找到目标绑定
2. 点击右侧删除图标
3. 确认解除绑定
4. 列表自动刷新

## 🎨 状态说明

### 绑定状态
| 状态 | 颜色 | 说明 |
|------|------|------|
| 生效中 | 绿色 | 绑定关系正常 |
| 已过期 | 灰色 | 绑定已超过有效期 |
| 已解除 | 红色 | 绑定关系已解除 |

### 角色说明
| 角色 | 图标颜色 | 说明 |
|------|---------|------|
| 发送者 | 橙色 | 我是发送者，对方是接收者 |
| 接收者 | 绿色 | 我是接收者，对方是发送者 |

## 📐 UI 状态管理

| 状态 | 显示内容 |
|------|---------|
| 加载中 | 骨架屏占位 + 加载指示器 |
| 空数据 | 空状态提示 |
| 有数据 | 绑定关系列表 |
| 错误 | 错误视图 + 重试按钮 |

---

## 🔧 技术实现

### 状态管理
- 使用 `BindingProvider` 管理所有绑定状态
- 使用 `Consumer` 监听状态变化
- 自动显示加载、错误、成功状态

### 初始化流程
1. 页面加载时，`BindingProvider` 自动加载数据
2. 调用 `loadBindings()` 加载绑定列表
3. 显示绑定数据和统计信息

### 错误处理
- 网络错误：显示错误视图，提供重试按钮
- 操作失败：显示 SnackBar 错误提示
- 自动清除错误状态
