# BindingTab 功能说明

## 📋 核心概念

### 手机号绑定（远程绑定）
- **用途**：建立长期或短期绑定关系的唯一方法
- **场景**：发送者知道对方手机号，远程建立绑定关系
- **流程**：输入对方 11 位手机号 → 发送绑定请求 → 接收者确认 → 建立关系
- **有效期设置**：
  - **永久绑定**：长期关系（如家庭成员）
  - **有限时间绑定**：自定义有效期（如旅游团导游带团 7 天）
  - 由发送者定义，过期后自动解除绑定
- **特点**：绑定后可随时查看对方位置、发送导航任务

### 二维码分享路线（临时分享）
- **用途**：一次性分享路线，**无需绑定关系**
- **场景**：面对面时，发送者生成路线二维码，多人可扫码接受同一路线
- **流程**：发送者规划路线 → 生成二维码 → 多人扫码 → 各自开始导航
- **特点**：
  - 每次规划路线生成一个二维码
  - 一个二维码可被多人扫描使用
  - 临时性、无需注册、无需绑定

## 🔒 位置共享权限控制

### 双向控制权
- **发送者**：可以随时允许/拒绝他人查看自己的位置
- **接收者**：可以随时允许/拒绝他人查看自己的位置

### 控制时机
- **导航开始前**：在规划路线后、导航启动前设置
- **导航进行中**：在导航过程中随时切换

### 权限选项
- **允许查看**：其他人可以看到实时位置
- **拒绝查看**：隐藏自己的位置信息

## 🔄 使用流程

### 发送者绑定接收者

1. **发送绑定请求**：
   - 点击"绑定新用户"
   - 输入对方的 11 位手机号
   - 可选：输入备注（如"给父亲的绑定"）
   - 点击"发送绑定请求"
   - 等待接收者确认

2. **接收者确认**：
   - 查看待确认的绑定请求
   - 确认或拒绝绑定
   - 确认后绑定关系生效，列表自动刷新

### 解除绑定

1. 在绑定列表中，找到要解除的绑定
2. 点击右侧的删除图标
3. 确认解除绑定
4. 解除成功，列表自动刷新

## 🎨 状态说明

### 绑定状态
- **生效中**（绿色）- 绑定关系正常，可以正常使用
- **已过期**（灰色）- 绑定已超过有效期
- **已解除**（红色）- 绑定关系已解除

### 角色说明
- **发送者**（橙色图标）- 我是发送者，对方是接收者
- **接收者**（绿色图标）- 我是接收者，对方是发送者

## ⚠️ 限制说明

### 发送者限制
- 最多绑定 **5** 个接收者
- 达到上限后，"绑定新用户"按钮会自动禁用
- 统计卡片显示"已达上限"警告

### 接收者限制
- 最多被 **3** 个发送者绑定
- 达到上限后，仍可以绑定新用户（作为发送者）
- 统计卡片显示"已达上限"警告

## 🔧 技术实现

### 状态管理
- 使用 `BindingProvider` 管理所有绑定状态
- 使用 `Consumer` 监听状态变化
- 自动显示加载、错误、成功状态

### 初始化流程
1. 页面加载时，`BindingProvider` 自动从 `ApiClient` 加载数据（Token 由 `SecureStorage` 自动提供）
2. 调用 `loadBindings()` 加载绑定列表
3. 显示绑定数据和统计信息

### 错误处理
- 网络错误：显示错误视图，提供重试按钮
- 操作失败：显示 SnackBar 错误提示
- 自动清除错误状态
