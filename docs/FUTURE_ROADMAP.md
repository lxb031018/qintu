## 1. 账号与手机号管理（高优先级）

### 1.1 换绑手机号
**场景**：用户更换了手机号码，但希望保留原有的绑定关系和历史数据。
**架构支持**：
- 当前用户 ID (`openid`) 为随机 UUID，不依赖手机号生成。
- **实现方案**：在数据库中更新用户的 `phone_number` 字段，并刷新“电话本”映射即可。
- **数据迁移**：由于 ID 不变，所有关联的绑定关系、任务记录都会自动跟随新用户手机号。

### 1.2 手机号解绑/多账号合并
**场景**：用户希望解绑旧手机号，或者将两个账号的数据合并。
**实现方案**：
- 增加“个人中心 - 账号安全”页面。
- 验证旧手机号 -> 输入新手机号 -> 验证新手机号 -> 更新数据库。

## 2. 云服务器迁移（中优先级）

### 2.1 从 CloudBase 迁移到自建服务器
**当前状态**：使用适配器模式（Adapter Pattern）隔离了网络层。
**实施步骤**：
1. 实现 `CustomServerAdapter`。
2. 在 `EnvironmentManager` 中配置新服务器地址。
3. App 端无需修改业务逻辑，只需切换环境配置。
