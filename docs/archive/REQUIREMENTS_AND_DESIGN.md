# 亲途 (Qintu) - 项目需求与技术文档

> **项目定位**：一款解决老年群体"导航难"痛点的 Flutter 移动应用  
> **核心理念**：通过"远程代操作"模式，将复杂的路径规划、路线修改等操作转移给发送者，接收者仅需"一键接受"即可享受导航服务

## 📖 项目简介

### 痛点分析

老年群体在使用传统导航应用时面临以下困难：

| 痛点 | 说明 |
|------|------|
| **操作复杂** | 输入目的地、选择路线、切换语音等步骤对接收者不友好 |
| **视力障碍** | 小字体、复杂界面难以阅读 |
| **认知负担** | 路线规划、交通方式选择等决策过程复杂 |
| **应急困难** | 走错路或计划变更时，接收者不知如何重新规划 |

### 解决方案

**亲途**采用"发送者代操作 + 接收者一键接受"模式：

- ✅ **发送者端（发送者）**：完整的路径规划、路线管理、位置监护功能
- ✅ **接收者端（接收者）**：极简界面，一键接受导航，语音引导

### 典型使用场景

```
场景：父亲想去北京站
1. 父亲打电话告诉儿子想去北京站（应用外沟通）
2. 儿子打开亲途App，显示父亲当前所在位置，输入"北京站"，亲途自动规划路线
3. 儿子点击"发送给父亲"
4. 父亲手机收到通知，点击"接受导航"
5. 父亲跟随语音导航前往北京站
6. 儿子可随时查看父亲位置，必要时可修改路线或结束导航
```

## 🔄 核心业务流程

### 流程概览

```
┌─────────────┐
│  沟通意图    │  ← 接收者通过微信/电话告知目的地（应用外流程）
└──────┬──────┘
       ▼
┌─────────────┐
│  远程规划    │  ← 发送者输入目的地，规划路线，发送给接收者
└──────┬──────┘
       ▼
┌─────────────┐
│  一键导航    │  ← 接收者手机收到通知，点击"接受"，开始语音导航
└──────┬──────┘
       ▼
┌─────────────┐
│  按需监护    │  ← 发送者可随时查看位置，忙时可退出，不影响接收者导航
└──────┬──────┘
       ▼
┌─────────────┐
│  远程干预    │  ← 发送者可修改路线、结束导航
└──────┬──────┘
       ▼
┌─────────────┐
│  行程结束    │  ← 到达目的地，导航自动结束
└─────────────┘
```

### 流程详解

| 阶段 | 操作方 | 关键动作 | 说明 |
|------|--------|---------|------|
| **1. 沟通意图** | 接收者 → 发送者 | 电话/微信告知目的地 | 应用外流程，不依赖 App |
| **2. 远程规划** | 发送者 | 输入目的地 → 预览路线 → 发送 | 发送者端完整操作 |
| **3. 一键导航** | 接收者 | 点击"接受" → 开始导航 | 极简操作，语音引导 |
| **4. 按需监护** | 发送者 | 查看位置 → 退出查看 | 按需触发，节省资源 |
| **5. 远程干预** | 发送者 | 修改路线 / 结束导航 | 应急处理能力 |
| **6. 行程结束** | 系统 | 到达目的地 → 自动结束 | 无需手动操作 |

## 🧩 功能模块详解

### 3.1 发送者端（发送者端）

#### 核心功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **路线规划与下发** | 输入目的地，预览路线，推送导航指令 | 🔴 P0 |
| **按需位置查看** | 默认不显示轨迹，点击后查看实时位置 | 🟡 P1 |
| **远程干预控制** | 修改路线、结束导航 | 🔴 P0 |

#### 功能详情

##### ① 路线规划与下发

```
操作流程：
1. 选择已绑定的接收者
2. 选择出行方式（驾车 / 公交 / 步行 / 骑行）- 由高德地图组件提供
3. 输入目的地名称
4. 预览路线（地图显示 + 距离/时间信息）
5. 点击【发送给接收者】
6. 系统推送导航指令到接收者手机
```

##### ② 按需位置查看

| 状态 | 行为 | 说明 |
|------|------|------|
| **默认状态** | 不显示实时轨迹 | 节省服务器资源和流量 |
| **点击查看** | 建立连接，显示位置与轨迹 | 按需触发 |
| **退出查看** | 断开位置流传输 | 接收者端不受影响 |

##### ③ 远程干预控制

**修改路线**：
```
场景：接收者走错路或计划变更
操作：发送者重新规划路线 → 再次发送
接收者端：弹出提示"发送者为您更新了路线，是否接受？"
```

**结束导航**：
```
场景：接收者已到达或需取消导航
操作：发送者点击【远程结束】
接收者端：导航自动停止，恢复待机界面
```

##### ② 导航执行与保活

**前台服务机制**：

| 平台 | 保活策略 | 说明 |

| **iOS** | `Background Modes` | 配置 Location, Audio，利用系统导航能力保持后台运行 |

## 🛠️ 技术选型与架构

### 技术栈

| 类别 | 技术选型 | 说明 |
|------|---------|------|
| **开发框架** | Flutter 3.x | 跨平台移动应用（Android + iOS） |
| **地图 SDK** | 高德地图 Flutter 插件 | 定位、地图渲染、路线规划、导航组件 |
| **状态管理** | Provider | 简单易用的全局状态管理 |
| **网络请求** | Dio | HTTP 客户端 |
| **实时通信** | WebSocket / MQTT | 位置实时传输、指令推送 |
| **本地存储** | flutter_secure_storage | 安全存储用户凭证 |

### 保活策略（关键）

#### Android 保活

```dart
// 使用 flutter_background_service
// 导航时创建前台服务
await FlutterBackgroundService().startService(
  androidConfig: AndroidConfiguration(
    autoStart: true,
    isForegroundMode: true,
    notificationTitle: '亲途导航进行中',
    notificationText: '距离终点：5.3km，预计12分钟',
  ),
);
```

#### iOS 保活

```xml
<!-- Info.plist 配置 -->
<key>UIBackgroundModes</key>
<array>
  <string>location</string>
  <string>audio</string>
</array>
```

### 通信架构

```
┌──────────────────────────────────────────────┐
│                  后端服务器                   │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│  │ REST API │  │ WebSocket│  │   MQTT   │  │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘  │
└───────┼─────────────┼─────────────┼─────────┘
        │             │             │
        ▼             ▼             ▼
   控制指令      实时位置      消息推送
  (开始/结束)   (位置上报)   (通知提醒)
```

| 通信方式 | 用途 | 触发时机 |
|---------|------|---------|
| **REST API** | 控制指令（开始、结束、修改路线） | 发送者操作时 |
| **WebSocket** | 实时位置传输 | 仅当发送者点击"查看位置"时 |
| **推送通知** | 新任务提醒、路线更新 | 接收到指令时 |

## 📊 数据结构设计

### 核心数据模型

#### User（用户）

```dart
class User {
  final String id;           // 用户 ID
  final String openid;       // 微信 OpenID
  final String phone;        // 手机号
  final String nickname;     // 昵称
  final String role;         // 角色：sender / receiver / both
  final DateTime createdAt;  // 创建时间
}
```

#### NavigationTask（导航任务）

```dart
class NavigationTask {
  final String id;                // 任务 ID
  final String senderId;          // 发送者 ID
  final String receiverId;        // 接收者 ID
  final String status;            // 状态：pending / active / finished / cancelled
  final String startName;         // 起点名称
  final String endName;           // 终点名称
  final double startLatitude;     // 起点纬度
  final double startLongitude;    // 起点经度
  final double endLatitude;       // 终点纬度
  final double endLongitude;      // 终点经度
  final Map routeData;            // 高德路线数据（JSON）
  final int distanceMeters;       // 距离（米）
  final int durationSeconds;      // 预计时间（秒）
  final DateTime createdAt;       // 创建时间
  final DateTime? startedAt;      // 开始时间
  final DateTime? finishedAt;     // 完成时间
}
```

#### UserBinding（绑定关系）

```dart
class UserBinding {
  final String id;                // 绑定 ID
  final String senderPhone;       // 发送者手机号
  final String receiverPhone;     // 接收者手机号
  final String status;            // 状态：pending / active / expired / revoked
  final String? remark;           // 备注（如：父亲、母亲）
  final DateTime createdAt;       // 创建时间
  final DateTime? activatedAt;    // 激活时间（接收者确认后）
}
```

**绑定流程（手机号绑定方案）**：

```
发送者端（发送者）：
1. 输入接收者（接收者）的手机号
2. 点击"发送绑定请求"
3. 系统向接收者推送绑定请求

接收者端（接收者）：
1. 收到绑定请求通知
2. 查看发送者手机号和姓名
3. 点击"确认绑定" → 绑定成功
   或点击"拒绝" → 拒绝请求
```

### 数据库表结构

| 表名 | 说明 | 关键字段 |
|------|------|---------|
| **users** | 用户表 | phone, nickname, role, created_at |
| **user_bindings** | 绑定关系表 | sender_phone, receiver_phone, status, remark |
| **navigation_tasks** | 导航任务表 | sender_id, receiver_id, status, route_data |
| **real_time_locations** | 实时位置表 | task_id, latitude, longitude, timestamp |
| **operation_logs** | 操作日志表 | user_id, action, details, created_at |

**绑定关系说明**：
- `sender_phone`: 发送者（发送者）手机号
- `receiver_phone`: 接收者（接收者）手机号
- 通过手机号查询用户，建立绑定关系
- 状态：`pending`（待确认）→ `active`（已激活）
