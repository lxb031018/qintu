# 云函数部署指南（手动上传）

> **说明**：由于云函数 HTTP 模式不会自动安装依赖，我们需要将包含 `node_modules` 的完整包上传到云端。
> **文件位置**：`D:\AllCodes\qintu\qintu-api-with-modules.zip`

---

## 🚀 部署步骤

### 第一步：登录控制台
1.  访问 [CloudBase 控制台](https://console.cloud.tencent.com/tcb)。
2.  进入环境：**`qintu-cloudebase-5f5bpuj13bc6467`**。

### 第二步：上传代码
1.  在左侧菜单点击 **"云函数"**。
2.  点击函数名称 **`qintu-api`** 进入详情页。
3.  点击顶部的 **"函数代码"** 标签。
4.  点击 **"部署"** 或 **"更新代码"** 按钮（通常在右上角）。
5.  在弹出的窗口中选择 **"本地上传 zip 包"**。
6.  点击选择文件，找到本机路径：
    `D:\AllCodes\qintu\qintu-api-with-modules.zip`
7.  点击 **"上传并部署"**。

### 第三步：重启实例（重要）
1.  上传完成后，回到函数详情页面。
2.  点击顶部的 **"函数配置"**。
3.  在底部或操作栏找到 **"重启实例"** 按钮并点击。
    *   *解释：重启可以确保云端加载新上传的依赖库和环境变量。*

---

## 🧪 验证测试

1.  部署完成后，在手机端重新运行应用（`flutter run`）。
2.  输入手机号并点击 **"获取验证码"**。
3.  **观察日志**：
    *   如果在终端看到 `[Auth] 请求发送验证码: +86 ...` 且没有报错，说明函数已正确加载。
    *   如果仍然失败，请检查云端函数的 **"运行日志"**，查看是否有 `Cannot find module` 错误。
