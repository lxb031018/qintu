# 文档自动更新规则

> 当 AI 执行代码重构或架构变更时，必须遵守此规则。

---

## 📋 何时需要更新文档？

### ✅ 必须更新（自动执行）

| 触发条件 | 更新哪些文档 | 示例 |
|---------|-------------|------|
| **新增/删除文件/目录** | `PROJECT_ARCHITECTURE.md` | 新增 `binding/widgets/` 目录 |
| **修改架构规则** | `QWEN.md` | 移除单例、添加 Provider |
| **新增/删除依赖** | `pubspec.yaml` + `QWEN.md` | 添加 `mocktail` |
| **重构导致行数变化 >30%** | `PROJECT_ARCHITECTURE.md` | `binding_page.dart` 742→233 行 |
| **修复 issue >3 个** | 两个文档都要更新 | 修复废弃导入、硬编码等 |
| **新增测试 >5 个** | `PROJECT_ARCHITECTURE.md` | 测试从 11→29 个 |

### ⚠️ 可选更新（询问用户）

| 触发条件 | 建议操作 |
|---------|---------|
| 修改常量/配置 | 更新架构文档的"待完善项" |
| 小范围代码优化 | 更新重构完成项列表 |
| 修复 1-2 个 issue | 可选择性更新 |

### ❌ 不需要更新

| 场景 | 原因 |
|------|------|
| 修复 UI 样式 | 不影响架构 |
| 修改文案/字符串 | 不影响结构 |
| 添加注释 | 不影响行为 |

---

## 🔧 更新执行流程

### 步骤 1: 识别变更类型

```
代码变更 → 判断影响范围 → 确定更新哪些文档
```

### 步骤 2: 更新 QWEN.md

**检查清单**：
- [ ] 架构规则是否变化？（Provider、单例、依赖注入）
- [ ] 是否有新的"禁止"规则？
- [ ] 是否有新的服务/管理器注册方式？

**更新位置**：
- `## 🏗️ 架构规则` 章节
- 添加新的子章节（如 `### Token 刷新`）

### 步骤 3: 更新 PROJECT_ARCHITECTURE.md

**检查清单**：
- [ ] 目录结构是否变化？
- [ ] 是否有新增/删除组件？
- [ ] 重构完成项是否需要补充？
- [ ] 待完善项是否需要同步？

**更新位置**：
- `## 📁 功能模块结构` 章节
- `## ✅ 重构完成项` 章节

### 步骤 4: 验证更新

```bash
# 确保代码无 issue
flutter analyze lib/

# 确保测试通过
flutter test
```

## 📝 更新格式规范

### QWEN.md 格式

```markdown
### 新功能/规则名称
- **实现位置**：`lib/xxx/yyy.dart`
- **使用方式**：如何调用/注册
- **禁止事项**：不应该做什么
- **注意事项**：特殊说明
```

### PROJECT_ARCHITECTURE.md 格式

```markdown
## ✅ 重构完成项

### 分类名称（日期）
1. ✅ 具体成果（文件/行数变化/提升百分比）
2. ✅ 另一个成果
```

## 🤖 AI 自动决策逻辑

```
if 代码变更涉及以下任一条件:
  ✓ 新增/删除文件或目录
  ✓ 架构模式变化（单例→Provider、Repository→Direct API）
  ✓ 行数变化 >30%
  ✓ 修复 issue >3 个
  ✓ 新增测试 >5 个
  
then:
  1. 自动更新 QWEN.md（架构规则）
  2. 自动更新 PROJECT_ARCHITECTURE.md（目录结构/重构历史）
  3. 运行 flutter analyze 验证
  4. 运行 flutter test 验证
  5. 向用户报告更新内容

else:
  不更新文档，继续正常流程
```

---

## 📌 重要提醒

1. **不要写历史总结** → 写"以后怎么做"的规则
2. **保持文档简洁** → 只记录 AI 需要知道的信息
3. **及时更新** → 代码改完立即更新文档
4. **验证后再报告** → 确保代码无 issue 再告诉用户

## 💡 示例：何时该更新？

### 场景 1: 重构 binding_page.dart ✅ 需要更新
- **触发**：742 行 → 233 行 (-69%)
- **更新**：
  - `PROJECT_ARCHITECTURE.md` → 目录结构 + 重构完成项
  - `QWEN.md` → 无需更新（架构规则未变）

### 场景 2: 移除 ThemeManager 单例 ✅ 需要更新
- **触发**：架构规则变化
- **更新**：
  - `QWEN.md` → 添加"主题管理"规则
  - `PROJECT_ARCHITECTURE.md` → 重构完成项

### 场景 3: 修复按钮颜色 ❌ 不需要更新
- **触发**：UI 样式修改
- **更新**：无
