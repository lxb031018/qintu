# 亲途 - 项目需求与技术文档
## 1. 项目简介
“亲途”是一款解决老年群体“导航难”痛点的 Flutter 移动应用。通过“远程代操作”模式，将复杂的路径规划、路线修改等操作转移给子女，长辈仅需“一键接受”即可享受导航服务。

## 2. 核心业务流程
沟通意图：长辈通过微信/电话告知子女想去的目的地（应用外流程）。
远程规划：子女打开“亲途”，输入目的地并规划路线，发送给长辈。
一键导航：长辈手机收到通知，点击“接受”后，App 挂载前台服务，开始语音导航。
按需监护：子女想查看时点击“查看位置”，查看长辈当前进度；忙时可随时退出，不影响长辈导航。
远程干预：
修改路线：子女发现长辈走错或计划变更，重新规划路线发送，长辈点击“接受”即可切换。
结束导航：子女可远程结束当前导航任务。
行程结束：长辈到达目的地，导航自动结束。
## 3. 功能模块详解
### 3.1 晚辈端（子女端）
路线规划与下发：
输入目的地，预览路线。
点击【发送给长辈】，推送导航指令。
按需位置查看：
默认不显示实时轨迹。
点击【查看实时位置】按钮时，建立连接并显示长辈当前位置与轨迹。
点击【退出】或切后台，断开位置流传输（长辈端不受影响），节省资源。
远程干预控制：
修改路线：在长辈导航过程中，子女可重新规划路线并再次发送，长辈端弹出提示“子女为您更新了路线，是否接受？”。
结束导航：点击【远程结束】，长辈端导航自动停止，恢复待机界面。
### 3.2 长辈端
极简主页：
大字体状态提示（“等待子女发送路线…” / “导航进行中”）。
导航执行与保活：
接收指令后，弹出硕大的【接受导航】弹窗。
前台服务机制：导航开始，App 自动开启前台服务，在通知栏常驻“导航中”状态，确保锁屏/后台时不被系统杀死，定位与语音持续工作。
支持接收子女发来的“新路线”或“结束指令”，并给出明确提示。
## 4. 技术选型与架构
开发框架：Flutter (跨平台)
地图 SDK：高德地图 SDK (Flutter 插件版)
集成：定位、地图渲染、导航组件。
保活策略 (关键)：
Android：使用 flutter_background_service 或原生 Foreground Service。导航时拉起通知栏常驻通知，利用前台权限保证进程存活。
iOS：配置 Background Modes (Location, Audio)，利用系统导航能力保持后台运行。
通信架构：
指令推送：通过后端服务器转发控制指令（改路线、结束、开始）。
实时位置：采用 WebSocket 或 MQTT。仅当晚辈点击“查看”时触发位置数据的高频上报与传输。
## 5. 数据结构设计简述
User: { id, role(elder/junior), bindCode }
NavigationTask: { id, juniorId, elderId, status(active/finished), startLoc, endLoc, routeData(高德路线数据) }

阶段二：地图与导航

实现：晚辈规划路线 -> 传给长辈 -> 长辈点击后调起高德导航组件。
阶段三：前台保活
重点攻克：实现 Android 前台服务，确保长辈锁屏后导航不停。
阶段四：监护与干预
实现“按需查看位置”逻辑。
实现“中途修改路线”和“远程结束”功能。
